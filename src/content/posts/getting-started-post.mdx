# Getting Started: Making a Post

<div className="text-gray-700 dark:text-neutral-300 text-lg italic mb-8">
  "The journey of a thousand readers begins with a single markdown file." - Content Creator Proverb
</div>

## Overview

Creating and publishing content on Temporal Flow is straightforward and flexible. This guide will walk you through the process of writing your first post, formatting it correctly, and making it available on your site.

<div className="card-base p-6 mb-6">
  <h3 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">The Basics</h3>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Temporal Flow uses a content-first approach where your posts are simply text files stored in a specific folder. The platform automatically processes these files and displays them on your site without any manual publishing steps.</p>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">All posts are written in either Markdown (.md) or MDX (.mdx) format, giving you powerful formatting options while keeping the content creation process simple and focused.</p>
</div>

<div className="card-base p-6 mb-6 bg-[oklch(0.98_0.025_var(--hue))] dark:bg-[oklch(0.2_0.025_var(--hue))]">
  <h3 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-300">Two Ways to Create Content</h3>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Temporal Flow offers two approaches to content creation:</p>
  
  <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
    <div className="bg-white dark:bg-neutral-800 p-4 rounded-lg">
      <h4 className="font-bold">Admin Panel Approach</h4>
      <p className="text-sm">Create and manage posts directly through the browser-based admin interface with GitHub integration for easy publishing.</p>
    </div>
    
    <div className="bg-white dark:bg-neutral-800 p-4 rounded-lg">
      <h4 className="font-bold">Developer Approach</h4>
      <p className="text-sm">Create markdown files directly in your code editor and manage them through traditional git workflows.</p>
    </div>
  </div>
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">This guide covers both methods, so you can choose the workflow that suits you best.</p>
</div>

## Method 1: Creating Posts with the Admin Panel

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Using the Browser-Based Admin</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">The admin panel provides a convenient interface for creating posts without having to directly edit files:</p>
  
  <ol className="list-decimal pl-5 space-y-2 text-gray-700 dark:text-neutral-300">
    <li>
      <strong>Access the admin panel</strong> by navigating to your site's <code>/admin</code> path and logging in
    </li>
    <li>
      <strong>Navigate to the Content tab</strong> and click "Create New Post"
    </li>
    <li>
      <strong>Fill in the metadata form</strong> with title, description, publication date, etc.
    </li>
    <li>
      <strong>Write your content</strong> using the Markdown editor
    </li>
    <li>
      <strong>Preview your post</strong> to see how it will look on your site
    </li>
    <li>
      <strong>Save and commit</strong> directly to your GitHub repository
    </li>
    <li>
      <strong>Trigger a deployment</strong> to publish your changes
    </li>
  </ol>
  
  <div className="bg-blue-50 dark:bg-blue-900/20 p-4 rounded-md mt-4 text-blue-800 dark:text-blue-200">
    <p className="font-bold">GitHub Integration</p>
    <p className="text-sm mt-1">
      With GitHub integration enabled, your post will be automatically committed to your repository when you save it, and you can trigger a rebuild directly from the admin panel to publish it.
    </p>
  </div>
</div>

## Method 2: Creating Posts with File Editing

### Step 1: Create Your File

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">File Creation</h4>
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Start by creating a new text file in the <code>content/posts/</code> directory of your Temporal Flow installation. You can organize files in subdirectories if you wish.</p>
  
  <div className="bg-gray-100 dark:bg-gray-800 p-4 rounded-md mb-4">
    <p className="font-mono text-sm">content/posts/my-first-post.mdx</p>
  </div>
  
  <p className="text-gray-700 dark:text-neutral-300">You can use either the .md extension for standard Markdown or .mdx if you want to include React/Svelte components in your content.</p>
</div>

### Step 2: Add Frontmatter

Every post begins with "frontmatter" - a YAML section at the top of the file that defines metadata about your post. This is enclosed between triple dashes `---`.

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Required Frontmatter</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">At minimum, you need to include:</p>
  
  ```yaml
  ---
  title: "My First Post"
  published: 2025-03-30
  description: "A brief description of what this post is about."
  ---
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">The <code>title</code> will be displayed as the main heading, <code>published</code> is the publication date (YYYY-MM-DD format), and <code>description</code> is used for SEO and preview snippets.</p>
</div>

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Optional Frontmatter</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">For more control, you can add these optional fields:</p>
  
  ```yaml
  ---
  # Basic metadata
  image: "/posts/my-post/feature.jpg"  # Featured image
  tags: [Tutorial, Guides, Beginners]   # Related topics
  category: "Documentation"             # Primary category
  draft: false                          # Set to true to hide from production
  lang: "en"                            # Content language
  
  # Custom author (override site defaults)
  avatarImage: "/authors/my-avatar.png"
  authorName: "Your Name"
  authorBio: "Brief author description"
  
  # Banner configuration options
  bannerType: "image"                   # "image", "video", or "timeline"
  
  # Timeline integration (if your post is also a timeline event)
  timelineYear: 2025
  timelineEra: "documentation"
  timelineLocation: "Temporal Flow"
  isKeyEvent: false                     # Highlight as a key event?
  ---
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">
    <strong>Note:</strong> When using the admin panel, these frontmatter fields are presented as form inputs with helpful labels and validation.
  </p>
</div>

### Step 3: Write Your Content

After the frontmatter, add your post content using Markdown syntax:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Markdown Formatting</h4>
  
  <div className="grid grid-cols-1 md:grid-cols-2 gap-4 mb-4">
    <div>
      <p className="font-bold mb-2">Markdown Code</p>
      <div className="bg-gray-100 dark:bg-gray-800 p-3 rounded-md h-full">
        <pre className="text-sm">
# Heading 1
## Heading 2

**Bold text** and *italic text*

- List item 1
- List item 2

[Link text](https://example.com)

![Image alt text](/path/to/image.jpg)

> Blockquote text
        </pre>
      </div>
    </div>
    
    <div>
      <p className="font-bold mb-2">Rendered Result</p>

        <h2 className="text-lg font-bold">Heading 2</h2>
        <p className="mb-2"><strong>Bold text</strong> and <em>italic text</em></p>
        <ul className="list-disc pl-5 mb-2">
          <li>List item 1</li>
          <li>List item 2</li>
        </ul>
        <p className="mb-2"><a href="#" className="text-blue-500 underline">Link text</a></p>
        <p className="mb-2">[Image would appear here]</p>
        <blockquote className="border-l-4 border-gray-300 pl-4 italic">Blockquote text</blockquote>
      </div>
    </div>
  </div>
</div>

## Step 4: Publishing Your Post

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Publishing Workflow</h4>
  
  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
    <div>
      <h5 className="font-bold mb-3">Admin Panel Publishing</h5>
      <ol className="list-decimal pl-5 space-y-2 text-gray-700 dark:text-neutral-300 text-sm">
        <li>Click "Save" to store your post</li>
        <li>Click "Commit to GitHub" to push changes to your repository</li>
        <li>Click "Deploy Changes" to trigger a site rebuild</li>
        <li>Wait for the deployment to complete (usually 1-2 minutes)</li>
        <li>Your post is now live on your site</li>
      </ol>
    </div>
    
    <div>
      <h5 className="font-bold mb-3">Developer Workflow Publishing</h5>
      <ol className="list-decimal pl-5 space-y-2 text-gray-700 dark:text-neutral-300 text-sm">
        <li>Save your file in the <code>content/posts/</code> directory</li>
        <li>Add the file to git: <code>git add .</code></li>
        <li>Commit your changes: <code>git commit -m "Add new post"</code></li>
        <li>Push to GitHub: <code>git push</code></li>
        <li>Wait for CI/CD to rebuild your site (if configured)</li>
      </ol>
    </div>
  </div>
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">In both workflows, Temporal Flow will automatically:</p>
  
  <ul className="list-disc pl-5 space-y-2 text-gray-700 dark:text-neutral-300">
    <li>Process your frontmatter metadata</li>
    <li>Convert your Markdown/MDX to HTML</li>
    <li>Apply your site's theme and styles</li>
    <li>Add the post to your site's feed</li>
    <li>Update search indexes and sitemaps</li>
  </ul>
</div>

## Example: Complete Post

Here's what a complete post file might look like:

```markdown
---
title: "Getting Started with Temporal Flow"
published: 2025-03-30
description: "Your guide to setting up and using Temporal Flow for the first time."
image: "/posts/getting-started/banner.jpg"
tags: [Tutorial, Guide, Setup]
category: "Documentation"
---

# Getting Started with Temporal Flow

Welcome to Temporal Flow! This guide will help you get started with your new content platform.

## Installation

First, clone the repository:

```bash
git clone https://github.com/yourusername/temporal-flow.git
```

## Configuration

Edit `config.ts` to customize your site:

```typescript
export const siteConfig: SiteConfig = {
  title: 'My Amazing Blog',
  subtitle: 'Thoughts, stories, and ideas',
  // ...more options
}
```

That's all you need to get started!
```

## Troubleshooting

        <li>Verify your frontmatter has the required fields and correct format</li>
