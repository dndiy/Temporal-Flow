## Core Concepts

Before diving into specific features, let's understand some core concepts that make Temporal Flow unique:

<div className="grid grid-cols-1 md:grid-cols-2 gap-4 mb-8">
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Content-First Approach</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      Temporal Flow uses a content-first approach where your posts are simple text files stored in specific folders. No databases, no complex CMSes - just plain text files that get transformed into beautifully formatted web content.
    </p>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Decentralized Connections</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      Connect with other Temporal Flow sites directly, with no central server or algorithm controlling what you see. Friend connections are private to you, and you control exactly what sites you want to follow.
    </p>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Browser-Based Admin</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      Configure and manage your site through a sleek, browser-based admin panel. No need to edit code files directly - make changes through a user-friendly interface from any device.
    </p>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">GitHub Integration</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      Update your site directly from the admin panel with built-in GitHub integration. Changes are committed to your repository and your site is automatically rebuilt - all without leaving your browser.
    </p>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Temporal Context</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      The timeline feature helps readers understand content in chronological context, turning isolated posts into an interconnected narrative that can be explored in multiple ways.
    </p>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Complete Ownership</h4>
    <p className="text-sm text-gray-700 dark:text-neutral-300">
      You host your Temporal Flow site wherever you want (GitHub Pages, Netlify, Vercel, etc.), meaning you completely own and control your content, design, and audience relationship.
    </p>
  </div>
</div>

## Quick Start

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">For the Impatient</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Want to dive right in? Here's a streamlined process to get your Temporal Flow site up and running quickly:</p>
  
  <ol className="list-decimal pl-5 space-y-3 text-gray-700 dark:text-neutral-300">
    <li>
      <strong>Set up your repository</strong>
      <p className="mt-1">Clone the Temporal Flow starter template and install dependencies:</p>
      <div className="bg-gray-100 dark:bg-gray-800 p-3 rounded-md mt-2 font-mono text-sm">
        git clone https://github.com/yourusername/temporal-flow.git<br />
        cd temporal-flow<br />
        pnpm install
      </div>
    </li>
    
    <li>
      <strong>Deploy to GitHub Pages</strong>
      <p className="mt-1">Push your repository to GitHub and enable GitHub Pages for your repository.</p>
    </li>
    
    <li>
      <strong>Access the Admin Panel</strong>
      <p className="mt-1">Navigate to your deployed site's <code>/admin</code> path and log in with the default password.</p>
    </li>
    
    <li>
      <strong>Connect GitHub Integration</strong>
      <p className="mt-1">Create a GitHub Personal Access Token with repository access and connect it in the admin panel.</p>
    </li>
    
    <li>
      <strong>Configure your site</strong>
      <p className="mt-1">Use the admin panel to customize your site name, description, navigation, and other settings.</p>
    </li>
    
    <li>
      <strong>Create and publish content</strong>
      <p className="mt-1">Add your posts using Markdown, commit changes to GitHub, and deploy directly from the admin panel.</p>
    </li>
  </ol>
  
  <div className="bg-blue-50 dark:bg-blue-900/20 p-4 rounded-md mt-4 text-blue-800 dark:text-blue-200">
    <p className="font-bold">Alternative: Developer Workflow</p>
    <p className="mt-1 text-sm">If you prefer a more traditional development approach, you can also edit files directly, run the local development server with <code>pnpm run astro dev</code>, and push changes to GitHub manually.</p>
  </div>
</div>

## Key Configuration Files

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Important Files to Know</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-3">While these files can all be managed through the admin panel, it's helpful to understand the underlying structure:</p>
  
  <div className="space-y-4">
    <div>
      <p className="font-bold"><code>src/config.ts</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Main configuration file for your site settings, including site name, description, social links, and navigation options.</p>
    </div>
    
    <div>
      <p className="font-bold"><code>src/config/timelineconfig.ts</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Configuration for the timeline feature, including eras, display options, and visualization settings.</p>
    </div>
    
    <div>
      <p className="font-bold"><code>src/config/community.config.ts</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Settings for your community page, including Discord integration and contact forms.</p>
    </div>
    
    <div>
      <p className="font-bold"><code>src/config/about.config.ts</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Configuration for your about page, including team information and contact details.</p>
    </div>
    
    <div>
      <p className="font-bold"><code>content/posts/</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Directory where all your blog posts are stored as .md or .mdx files.</p>
    </div>
    
    <div>
      <p className="font-bold"><code>content/friends/</code></p>
      <p className="text-gray-700 dark:text-neutral-300">Directory containing your friend connection files for the Friend Content Sharing system.</p>
    </div>
  </div>
</div>
