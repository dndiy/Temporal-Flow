## Adding Friends

Once logged in, you can start connecting with other Temporal Flow sites.

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Step-by-Step Friend Addition</h4>
  
  <ol className="list-decimal pl-5 space-y-3 text-gray-700 dark:text-neutral-300">
    <li>
      <strong>Navigate to the Friends page</strong>
      <p className="mt-1">Click on the Settings button in the navigation bar and select "Friends" from the dropdown menu.</p>
    </li>
    
    <li>
      <strong>Add a new friend</strong>
      <p className="mt-1">On the Friends page, you'll see a form to add a new friend connection.</p>
      <div className="bg-gray-100 dark:bg-gray-800 p-3 rounded-md mt-2">
        <p className="text-sm mb-2">Required information:</p>
        <ul className="list-disc pl-5 text-sm">
          <li><strong>Friend Name</strong>: The name of the blog or site owner</li>
          <li><strong>Site URL</strong>: The complete URL to the friend's Temporal Flow site</li>
        </ul>
      </div>
    </li>
    
    <li>
      <strong>Click "Add Friend"</strong>
      <p className="mt-1">The system will validate the URL and attempt to establish a connection with the site.</p>
      <p className="mt-1">If successful, the site will be added to your temporary friends list, displayed below the form.</p>
    </li>
    
    <li>
      <strong>Save the friend connection permanently</strong>
      <p className="mt-1">To make this connection permanent, click the "Save Friend" button next to the temporary friend entry.</p>
      <p className="mt-1">You'll be prompted to save a markdown file - this is your friend connection file.</p>
      <p className="mt-1">Save this file in the <code>content/friends/</code> folder of your site.</p>
    </li>
  </ol>
</div>

### Understanding Friend Connection Files

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Friend Connection Files</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">When you save a friend connection, a markdown file is created with the following structure:</p>
  
  ```markdown
  ---
  friendName: "Example Blog"
  siteUrl: "https://example-blog.com"
  avatarUrl: "/friends/example-avatar.png"
  bio: "A great blog about interesting topics"
  addedOn: "2025-03-30"
  ---
  
  # Connection to Example Blog
  
  Added this connection on March 30, 2025. This blog focuses on technology tutorials.
  
  ## Notes
  
  - Great resource for JavaScript content
  - Updates weekly on Fridays
  - Owner: Jane Smith
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">These files are stored in the <code>content/friends/</code> directory and will persist until manually deleted. Each file represents one friend connection.</p>
</div>

## Technical Details: How It Works

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Behind the Scenes</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">The Friend Content Sharing system works through a special file called <code>friend-content.json</code> that's automatically generated in each Temporal Flow site. This file contains information about your blog and your recent posts.</p>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">When you add a friend, your site attempts to access their <code>friend-content.json</code> file to retrieve their content. For this to work across different domains, proper CORS headers need to be set up.</p>
  
  <div className="bg-gray-100 dark:bg-gray-800 p-4 rounded-md">
    <p className="font-bold mb-2">GitHub Pages CORS Setup</p>
    <p className="text-sm mb-2">Add a file named <code>_headers</code> to your repository root:</p>
    
    <pre className="text-xs bg-white dark:bg-gray-900 p-3 rounded-md">
# Set CORS headers for the friend-content.json file
/friend-content.json
  Access-Control-Allow-Origin: *
  Access-Control-Allow-Methods: GET
  Access-Control-Allow-Headers: Content-Type
    </pre>
  </div>
</div>

## Best Practices for Friend Connections

<div className="grid grid-cols-1 md:grid-cols-2 gap-4 mb-8">
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Do's</h4>
    <ul className="list-disc pl-5 text-sm text-gray-700 dark:text-neutral-300">
      <li>Connect with blogs in your niche or interest area</li>
      <li>Regularly sync content to see the latest posts</li>
      <li>Ask permission before adding someone as a friend</li>
      <li>Share your site URL with other Temporal Flow users</li>
      <li>Provide descriptive names and bios for friends</li>
    </ul>
  </div>
  
  <div className="card-base p-4">
    <h4 className="font-bold mb-2">Don'ts</h4>
    <ul className="list-disc pl-5 text-sm text-gray-700 dark:text-neutral-300">
      <li>Add too many friends (can impact performance)</li>
      <li>Expect real-time updates (content requires manual sync)</li>
      <li>Share friend connection files with others</li>
      <li>Modify friend content (view only)</li>
      <li>Forget to set up CORS headers on your site</li>
    </ul>
  </div>
</div>

## Troubleshooting

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Common Connection Issues</h4>
  
  <div className="space-y-4">
    <div>
      <p className="font-bold">Can't connect to friend site?</p>
      <ul className="list-disc pl-5 text-gray-700 dark:text-neutral-300">
        <li>Verify the site is a Temporal Flow site</li>
        <li>Check that the URL is correct and includes the protocol (https://)</li>
        <li>Ensure the site has the proper CORS headers configured</li>
        <li>Confirm the site is accessible (not down for maintenance)</li>
      </ul>
    </div>
    
    <div>
      <p className="font-bold">Friend content not appearing?</p>
      <ul className="list-disc pl-5 text-gray-700 dark:text-neutral-300">
        <li>Make sure you're logged in as an administrator</li>
        <li>Try syncing the content manually from the Friends page</li>
        <li>Check that the friend has published content recently</li>
        <li>Verify the friend connection file is in the correct location</li>
      </ul>
    </div>
  </div>
</div>

## Privacy Considerations

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Understanding Privacy</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">The Friend Content Sharing system is designed with privacy in mind:</p>
  
  <ul className="list-disc pl-5 space-y-2 text-gray-700 dark:text-neutral-300">
    <li>Your visitors never see your friend connections or their content</li>
    <li>Friend connections are always one-way (adding a friend doesn't give them access to your content)</li>
    <li>Friend data is stored locally in your site files</li>
    <li>No personal data beyond what's publicly available is shared</li>
    <li>You control exactly which sites you connect with</li>
  </ul>
</div>

## Conclusion

The Friend Content Sharing system transforms Temporal Flow from a standalone blog into a hub for connected content while maintaining your privacy and control. By building connections with other content creators, you create a personalized feed of content that's relevant to your interests and only visible to you.

As more Temporal Flow sites join the network, the value of this decentralized approach to content sharing will continue to grow, offering an alternative to centralized social platforms that puts creators in control.
