## How Timeline Integration Works

### Adding Content to the Timeline

Any blog post can become a timeline event by adding specific timeline metadata in the frontmatter:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Timeline Frontmatter Fields</h4>
  <p className="text-gray-700 dark:text-neutral-300 mb-4">To add a post to the timeline, include these fields in your post's frontmatter:</p>
  
  ```yaml
  ---
  # Required timeline fields
  timelineYear: 2023              # The year for timeline placement
  timelineEra: "development"      # The era classification 
  
  # Optional timeline fields
  timelineLocation: "GitHub"      # Location for map view
  isKeyEvent: true                # Highlight as a key event
  ---
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">Each post with these fields will automatically appear as an event in your site's timeline visualization.</p>
</div>

### Creating Era Classifications

Eras help organize your timeline into logical periods, making it easier for readers to understand the overall structure:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Defining Eras</h4>
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Eras are defined in <code>TimelineConfig.ts</code> with properties like:</p>
  
  ```typescript
  export const defaultEraConfig: EraConfigMap = {
    'planning': {
      displayName: 'Planning Phase',
      startYear: 2020,
      endYear: 2021,
      color: 'blue'
    },
    'development': {
      displayName: 'Development Phase',
      startYear: 2021,
      endYear: 2024,
      color: 'green'
    },
    'release': {
      displayName: 'Release Phase',
      startYear: 2024,
      endYear: 2025,
      color: 'purple'
    }
  }
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">Each era gets a distinct color and date range, making it easy to visually differentiate periods in your timeline.</p>
</div>

## Using Timelines in Your Content

### Timeline as a Banner

One of the most powerful uses of the timeline is as an interactive banner at the top of a page:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Timeline Banner Configuration</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">To use a timeline as your page banner, add this to your frontmatter:</p>
  
  ```yaml
  ---
  bannerType: "timeline"
  bannerData:
    category: "Project History"
    startYear: 2020
    endYear: 2025
    background: "/path/to/background.jpg"
    height: "70vh"  # Optional custom height
  ---
  ```
  
  <p className="text-gray-700 dark:text-neutral-300 mt-4">This creates an interactive timeline with all events from the specified category as your page header.</p>
</div>

### Timeline Page

You can also create a dedicated timeline page to showcase all your timeline content:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Creating a Timeline Page</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">To create a dedicated timeline page:</p>
  
  1. Create a new file at `src/pages/timeline.astro`
  2. Import the TimelinePage component
  3. Configure the timeline settings
  
  ```jsx
  ---
  import TimelinePage from "../components/TimelinePage.astro";
  import MainLayout from "../layouts/MainLayout.astro";
  
  const pageTitle = "Project Timeline";
  ---
  
  <MainLayout title={pageTitle}>
    <TimelinePage 
      title={pageTitle}
      description="The complete chronology of our project development"
      startYear={2020}
      endYear={2025}
      defaultEra="all"
    />
  </MainLayout>
  ```
</div>

## Understanding Timeline Elements

The timeline uses several visual indicators to help readers understand its contents:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Visual Indicators</h4>
  <ul className="list-disc pl-5 space-y-2 text-gray-700 dark:text-neutral-300">
    <li><strong>Event Nodes</strong>: Events appear as interactive nodes with colors indicating their era</li>
    <li><strong>Node Size and Brightness</strong>: Larger, brighter nodes represent key events that have the <code>isKeyEvent: true</code> property</li>
    <li><strong>Glow Effects</strong>: Selected events have a distinctive glow to highlight your current focus</li>
    <li><strong>Event Cards</strong>: Information cards show details when you select an event</li>
    <li><strong>Timeline Line</strong>: The central line represents the flow of time (horizontal on desktop, vertical on mobile)</li>
    <li><strong>Era Badges</strong>: Colored tags indicate which era an event belongs to</li>
    <li><strong>Map Pins</strong>: In Map view, events with location data appear as pins on the map</li>
  </ul>
</div>

## Timeline Configuration

To customize your timeline's appearance and behavior, you can modify the configuration:

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">TimelineConfig.ts</h4>
  
  <p className="text-gray-700 dark:text-neutral-300 mb-4">Edit <code>TimelineConfig.ts</code> to adjust timeline settings:</p>
  
  ```typescript
  export const timelineConfig = {
    // Visual settings
    nodeSize: 16,          // Base size of event nodes
    keyEventScale: 1.5,    // How much larger key events appear
    dateFormat: 'YYYY',    // Format for date display
    
    // Interaction settings
    animationDuration: 500,  // Animation speed in ms
    zoomLevels: {
      min: 0.5,            // Minimum zoom level
      max: 3,              // Maximum zoom level
      default: 1           // Default zoom level
    },
    
    // Default view settings
    defaultView: 'timeline',  // 'timeline', 'list', 'tree', or 'map'
    defaultEra: 'all',        // Which era to show by default
    
    // Mobile adaptations
    mobileBreakpoint: 768,    // Screen width to trigger mobile view
    mobileOrientation: 'vertical'  // Timeline orientation on mobile
  };
  
  // Era definitions
  export const defaultEraConfig = {
    // Era definitions as shown earlier
  };
  ```
</div>

## Advanced Timeline Tips

<div className="card-base p-6 mb-6">

    </li>
    <li>
      <strong>Time Precision</strong>: For more precise timelines, use <code>timelineDate</code> instead of <code>timelineYear</code> with ISO date format
    </li>
    <li>
      <strong>Connected Events</strong>: Use <code>relatedEvents</code> in frontmatter to link timeline events to each other
    </li>
  </ul>

## Troubleshooting

<div className="card-base p-6 mb-6">
  <h4 className="font-bold text-xl mb-4 text-gray-700 dark:text-neutral-400">Common Timeline Issues</h4>
  
  <div className="space-y-4">
    <div>
      <p className="font-bold">Posts not appearing in timeline?</p>
      <ul className="list-disc pl-5 text-gray-700 dark:text-neutral-300">
        <li>Check that <code>timelineYear</code> and <code>timelineEra</code> are defined in frontmatter</li>
        <li>Verify the era name matches one defined in <code>TimelineConfig.ts</code></li>
        <li>Ensure the post is not in draft mode (<code>draft: true</code>)</li>
        <li>Rebuild your site if you're in production mode</li>
      </ul>
    </div>
    
    <div>
      <p className="font-bold">Performance issues with large timelines?</p>
      <ul className="list-disc pl-5 text-gray-700 dark:text-neutral-300">
        <li>Use category filtering to limit the number of displayed events</li>
        <li>Consider splitting very large timelines into multiple smaller ones</li>
        <li>Reduce image sizes in event cards to improve load times</li>
        <li>Use the list view for timelines with hundreds of events</li>
      </ul>
    </div>
  </div>
